Rule frontmatter fields (all required unless noted):
- name: string (derived from filename if omitted — prefer explicit)
- severity: "block" | "warn"
- triggers: list, always [PreToolUse] for v0.1.0
- matches: list of tool names (e.g. [Bash], [Edit, Write]), or [*] for wildcard
- pattern: regex string (quote if it contains special chars)
- priority: integer (higher wins; 100 reserved for most-dangerous)
- anchored: bool, optional, default true — for substring matches use `anchored: false`
- message: optional; body text after frontmatter is used as the user-facing message

Example shape (verbatim from tests/fixtures/rules/block-rm-rf.md):
```
---
severity: block
triggers: [PreToolUse]
matches: [Bash]
pattern: 'rm\s+-rf'
priority: 100
anchored: false
---
Never run rm -rf — catastrophic data loss.
```

Parser is strict: invalid rules fail loudly to stderr (RULES-02).
ASCII-only in rule bodies (project rule — cp1252-safe).
